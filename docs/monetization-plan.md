# 명운판 광고 기반 유지 보완 계획

작성일: 2026-03-25
대상 서비스: `https://saju-wheat.vercel.app`

## 1. 한 줄 방향

명운판은 `정확한 무료 계산기`를 넘어, `무료로 읽고 머무르게 만드는 명리 콘텐츠 서비스`가 되어야 해요.

이 서비스는 광고만으로 유지하는 것을 목표로 해요.
따라서 가장 중요한 건 아래 4가지예요.

1. 계산 직후 바로 이탈하지 않게 만들기
2. 한 세션에서 더 많은 읽기 흐름을 만들기
3. 검색 유입이 계속 쌓이는 구조 만들기
4. 계정 없이도 다시 들어올 이유 만들기

중요한 전제는 분명해요.

- 개인화 서비스는 하지 않아요
- 회원가입, 계정 저장, 추천 알고리즘은 고려하지 않아요
- 대신 계산 결과, 탭 주제, 기사 주제라는 `현재 맥락`만 활용해요

---

## 2. 이번 계획의 핵심 원칙

### 원칙 1. 광고는 기능이 아니라 결과예요

광고를 많이 붙인다고 돈이 되는 게 아니에요.
사용자가 더 오래 읽고, 더 많이 이동하고, 다시 들어오게 만드는 제품 구조가 먼저예요.

### 원칙 2. AI는 비용 항목이 아니라 체류시간 장치예요

명운판에서 AI는 "있으면 좋은 부가기능"이 아니에요.
계산 결과를 사람이 이해 가능한 문장으로 바꾸고, 다음 클릭을 만들고, 콘텐츠 소비로 넘기는 핵심 장치예요.

### 원칙 3. 개인화 없이도 깊이는 만들 수 있어요

이 서비스는 사용자의 계정이나 장기 프로필 없이도 충분히 깊은 흐름을 만들 수 있어요.
방법은 아래 3가지예요.

1. 계산 직후 AI 요약 제공
2. 탭별 정적 후속 질문 제공
3. 주제별 정적 관련 글 연결

### 원칙 4. 광고는 오인 클릭을 만들면 안 돼요

광고는 버튼처럼 보이면 안 되고, AI 질문 카드나 CTA와 붙어 있으면 안 돼요.
광고는 항상 하나의 읽기 블록이 끝난 뒤에 나와야 해요.

- 탭별 정적 질문 세트
- 주제별 정적 관련 글 추천
- 오늘/이번 주 읽기형 진입점

## 4. 현재 상태 진단

### 이미 있는 것

- 홈, 폼, 결과에 광고 슬롯이 이미 있어요
- 결과 페이지에 AI 후속 흐름 패널이 있어요
- 탭별 후속 질문과 관련 글 추천이 있어요
- `오늘의 AI 운세` 진입이 있어요
- GTM `dataLayer` 기반 이벤트 푸시가 일부 있어요
- 다국어 라우팅과 기사/가이드 구조가 있어요

### 아직 부족한 것

- AI가 아직 독립적인 읽기 흐름이 아니라 모달 소비에 가까워요
- 기사 페이지가 검색 유입용이긴 하지만 광고 소비면이 약해요
- 기사 수가 아직 얇아서 검색 규모를 만들기 어려워요
- 퍼널 대시보드가 없어서 어떤 흐름이 돈이 되는지 판단이 어려워요
- `오늘의 AI 운세`가 편의 기능 수준이지 재방문 루프 수준은 아니에요
- 문서 기준과 실제 구현 상태가 조금 어긋나 있어요

## 5. 목표 사용자와 행동

### 핵심 사용자

- 무료 사주/운세를 검색해서 들어오는 모바일 사용자
- 표보다 쉬운 해석을 선호하는 사용자
- 계산 결과만 보고 끝내지 않고 연애, 직업, 재물 같은 질문을 더 보고 싶은 사용자
- 계정 없이 바로 소비하고 나가는 사용자를 포함한 가벼운 방문자

### 원하는 사용자 행동

1. 검색 또는 직접 유입
2. 계산
3. 결과 요약 확인
4. AI 한 줄 요약 확인
5. 후속 질문 클릭
6. 관련 글 이동
7. 다른 체계나 다른 주제 읽기
8. 오늘/이번 주 읽기 흐름으로 재방문

## 6. 제품 구조 목표

현재보다 아래 흐름이 훨씬 선명해야 해요.

```text
검색/랜딩
-> 계산
-> 결과 요약
-> AI 한 줄 요약
-> AI 상세 해석
-> 후속 질문
-> 관련 글
-> 다른 계산/다른 체계 보기
-> 오늘/이번 주 읽기
```

결과 페이지는 더 이상 `표를 보여주는 끝 페이지`가 아니에요.
명운판의 핵심 소비 허브가 되어야 해요.

## Phase 1. 결과 페이지 흐름 강화

기간: 1주

목표:
- 계산 직후 이탈을 줄여요
- 결과 페이지를 `표 -> 해석 -> 다음 읽기` 구조로 고정해요

할 일:
1. AI 해석 CTA를 결과 영역의 메인 액션으로 더 강하게 재배치
2. AI 한 줄 요약의 가시성 강화
3. 후속 질문 카드 우선순위 재정리
4. 관련 글 추천 노출 순서 조정
5. 다른 체계 보기 CTA 추가
6. 결과 하단 광고 위치를 읽기 흐름 기준으로 재정렬

성공 기준:
- 계산 완료 후 AI 진입률 상승
- 후속 질문 클릭률 상승
- 결과 페이지 이탈률 감소

주의:
- 광고는 AI 요약보다 먼저 나오지 않아요
- 광고는 후속 질문 버튼 묶음 안에 들어가지 않아요

---

## Phase 2. AI를 연속 소비 흐름으로 바꾸기

기간: 1~2주

목표:
- AI를 단발성 모달 기능이 아니라 연속 소비 흐름으로 바꿔요

할 일:
1. AI 해석 진입 경험을 더 자연스럽게 만들기
2. 후속 질문 클릭 시 같은 맥락에서 연속 소비가 이어지도록 설계
3. 탭별 정적 질문 세트 확장
4. `사주`, `자미두수`, `점성술`별 질문 조합 보완
5. AI 결과 바로 아래 관련 읽을거리 연결 강화

설계 원칙:
- 개인화 없이 탭 기준 정적 매핑 사용
- 질문은 짧고 쉬운 언어 사용
- 전문용어보다 사용자의 질문 언어 우선

성공 기준:
- AI 완료율 상승
- AI 후속 질문 클릭률 상승
- AI 이후 관련 글 클릭률 상승

## Phase 3. 콘텐츠 엔진 보강

기간: 2~4주

목표:
- 검색 유입을 키우고 계산/AI 소비로 연결해요

전략:
- 기사 수를 무작정 늘리지 않아요
- 먼저 전환이 강한 클러스터를 잡아요
- 한 글이 검색 유입과 계산 전환 둘 다 담당해야 해요

1차 우선 클러스터:
1. 연애/관계
2. 직업/재물
3. 출생시간 모름

각 클러스터 구성:
- 입문글 1개
- 실전 해석글 2개
- FAQ형 글 1개
- 계산 연결 글 1개

1차 목표:
- 클러스터 3개
- 총 15개 수준의 탄탄한 글

각 글의 필수 구성:
1. 주제 설명
2. 자주 묻는 질문
3. 계산 CTA
4. AI 질문 CTA
5. 관련 글 3개

성공 기준:
- 자연 검색 세션 증가
- 기사 -> 계산 전환율 측정 가능 상태
- 기사 -> AI 전환 흐름 확보

성공 기준:

## 9. KPI

운영 KPI는 최소한 아래 5개로 고정해요.

1. 계산 완료율
2. AI 진입률
3. AI 후속 질문 클릭률
4. 기사 클릭률
5. 세션당 페이지 수

보조 KPI:
- 평균 참여 시간
- 기사 -> 계산 전환율
- 기사 -> AI 전환율
- 오늘/이번 주 읽기 클릭률
- 결과 페이지 이탈률

## 10. 성공 기준

## 13. 한 줄 결론

명운판은 `개인화 서비스`가 아니라 `광고로 유지되는 무료 명리 미디어`로 가야 해요.
핵심은 사용자별 맞춤이 아니라, 누구에게나 자연스럽게 이어지는 `계산 -> AI -> 읽기 -> 재방문` 흐름을 만드는 거예요.
