# 명운판 광고-only 실행 PRD

작성일: 2026-03-25  
버전: v2  
상태: 실행 준비  
대상 서비스: `https://saju-wheat.vercel.app`

## 1. 문서 목적

이 문서는 명운판을 `광고만으로 유지 가능한 무료 명리 미디어`로 발전시키기 위한 실행용 PRD예요.

이 문서는 아이디어 정리 문서가 아니에요.
아래 4가지를 바로 결정하기 위한 문서예요.

1. 무엇을 만들지
2. 무엇을 만들지 않을지
3. 어떤 순서로 구현할지
4. 완료를 무엇으로 판단할지

---

## 2. 한 줄 정의

명운판은 `사주 계산기`가 아니라 `계산 -> AI -> 콘텐츠 -> 재방문` 흐름을 가진 광고 기반 무료 명리 서비스예요.

---

## 3. 절대 고정 전제

### 포함

- 광고 수익 모델
- 계산 결과 기반 AI 해석
- 탭별 정적 후속 질문
- 주제별 정적 관련 글 연결
- 로컬 기록 기반 다시 보기
- 오늘/이번 주 읽기형 재방문 진입

### 제외

- 회원가입
- 로그인
- 개인 프로필
- 서버 저장
- 행동 기반 추천
- 개인화 홈
- 개인 맞춤 광고
- 구독/프리미엄
- 커뮤니티 기능

이 PRD의 모든 구현은 위 전제를 어기면 안 돼요.

---

## 4. 해결하려는 문제

### 현재 문제

1. 사용자가 계산 결과를 본 뒤 빠르게 이탈해요
2. AI 해석은 있지만 연속 소비 흐름이 약해요
3. 기사/가이드가 검색 유입용으로는 존재하지만 수익화 흐름이 약해요
4. 무엇이 실제로 돈이 되는 흐름인지 측정 체계가 약해요

### 사업 문제

광고-only 모델에서 핵심은 광고 수를 늘리는 게 아니에요.
아래를 동시에 늘려야 해요.

1. 세션 길이
2. 세션당 페이지 수
3. 검색 유입
4. 재방문 이유

## 5. 제품 목표

### 비즈니스 목표

- 광고만으로 운영 가능한 트래픽 구조 만들기
- 홈 편중이 아닌 결과/기사 중심 광고 소비 구조 만들기
- 검색 유입과 내부 소비 흐름을 연결하기

### 제품 목표

- 결과 페이지 이탈 줄이기
- AI 진입과 후속 질문 소비 늘리기
- 기사에서 계산과 AI로 이어지는 흐름 만들기
- 개인화 없이 재방문 루프 만들기

### 핵심 KPI

1. 계산 완료율
2. AI 진입률
3. AI 후속 질문 클릭률
4. 기사 클릭률
5. 세션당 페이지 수

### 보조 KPI

- 평균 참여 시간
- 결과 페이지 이탈률
- 기사 -> 계산 전환율
- 기사 -> AI 전환율
- 오늘/이번 주 읽기 클릭률

## 6. 대상 사용자

### 핵심 사용자

- `무료 사주`, `무료 운세`, `출생시간 모를 때 사주`를 검색하는 모바일 사용자
- 표보다는 쉬운 문장형 해석을 선호하는 사용자
- 계산 결과를 본 뒤 연애/직업/재물 해석을 더 보고 싶은 사용자
- 계정 없이 빠르게 소비하고 나가는 가벼운 사용자

### 비대상 사용자

- 장기 프로필 관리가 필요한 사용자
- 개인별 맞춤 대시보드를 기대하는 사용자
- 유료 상담 서비스나 정기 구독을 기대하는 사용자

## 7. 목표 사용자 흐름

### 메인 흐름

```text
검색/직접 유입
-> 계산
-> 결과 요약
-> AI 한 줄 요약
-> AI 상세 해석
-> 후속 질문
-> 관련 글
-> 다른 탭/다른 계산
-> 오늘/이번 주 읽기
```

### 기사 흐름

```text
검색 유입
-> 기사 읽기
-> 본문 중 계산 CTA
-> 결과 확인
-> AI 해석 진입
-> 후속 질문
-> 관련 글
```

### 재방문 흐름

```text
홈 진입
-> 오늘의 AI 운세 / 이번 주 읽기
-> 로컬 기록 재사용 또는 새 계산
-> AI 해석
-> 관련 글
```

## 11. 상세 실행 단계

## Stage 1. 기준선 고정

기간: 2~3일

작업:
1. KPI와 이벤트 확정
2. 광고 배치 금지 규칙 확정
3. 현재 구현 상태와 문서 차이 정리

완료 조건:
- KPI 문서가 있다
- 이벤트 목록이 확정됐다
- 개인화 제외 원칙이 문서에 반영됐다

## Stage 2. 결과 페이지 개편

기간: 1주

작업:
1. 결과 페이지에서 AI 진입 영역 강화
2. 후속 질문 카드 개선
3. 관련 글과 다음 행동 CTA 재정렬
4. 광고 위치 재조정

완료 조건:
- 결과 페이지가 `결과 -> AI -> 질문 -> 글 -> 광고` 순서를 가진다
- AI 진입점이 하나의 메인 흐름으로 보인다
- 광고가 질문 버튼과 붙어 있지 않다

## Stage 3. AI 소비 흐름 강화

기간: 1~2주

작업:
1. 탭별 질문군 정리
2. 질문 수 5개 수준 확장
3. AI 결과 이후 다음 이동 경로 명확화

완료 조건:
- 각 탭마다 후속 질문군이 고정된다
- 후속 질문이 중복 없이 역할별로 나뉜다
- AI 이후 관련 글 클릭 경로가 명확하다

## Stage 4. 기사 페이지 수익화

기간: 1주

작업:
1. 광고 3지점 배치
2. 계산 CTA 삽입
3. AI 질문 CTA 삽입
4. 관련 글 보강

완료 조건:
- 기사에서 계산기로 가는 CTA가 상단 또는 중단에 있다
- 기사에 3개의 광고 소비면이 있다
- 광고가 본문 링크처럼 보이지 않는다

## Stage 5. 콘텐츠 클러스터 1차 확장

기간: 2~4주

1차 클러스터:
1. 연애/관계
2. 직업/재물
3. 출생시간 모름

각 클러스터 작업:
1. 입문글 1개
2. 실전글 2개
3. FAQ형 글 1개
4. 계산 연결형 글 1개

완료 조건:
- 총 15개 수준의 기사 초안이 있다
- 각 글에 계산 CTA와 AI 질문 CTA가 있다
- 관련 글 연결이 있다

## Stage 6. 재방문 루프

기간: 1주

작업:
1. 오늘의 AI 운세 보강
2. 이번 주 읽기 추가
3. 홈 진입점 강화

완료 조건:
- 홈에서 일간/주간 읽기 CTA가 보인다
- 로컬 기록이 있으면 재진입이 가능하다
- 기록이 없으면 계산으로 자연스럽게 연결된다

## Stage 7. 측정/튜닝

기간: 2~3일

작업:
1. 이벤트 검증
2. 퍼널 확인
3. 광고/CTA 배치 점검
4. KPI 기준 리포트 템플릿 작성

완료 조건:
- 이벤트가 실제로 들어온다
- 퍼널이 보인다
- 첫 주 운영 리포트를 뽑을 수 있다
