# 명운판 광고 전용 수익화 PRD

작성일: 2026-03-21  
대상 서비스: `https://saju-wheat.vercel.app/ko/`

## 1. 프로젝트 개요

### 서비스명
- **명운판 AI Ad Engine**
- 한글 설명: AI 체류시간을 광고 수익으로 바꾸는 명운판

### 한 줄 설명
- 사주팔자, 자미두수, 서양 점성술 계산 결과를 AI가 연속적으로 해석해 주고, 그 흐름 안에서 광고와 콘텐츠 소비를 늘려 광고 수익을 키우는 서비스예요.

### 해결하는 문제
- 계산기형 운세 서비스는 한 번 계산하고 바로 나가기 쉬워요.
- 광고-only 모델에서 가장 큰 문제는 비용보다 **짧은 세션**이에요.
- 명운판은 이미 계산 정확도, 다국어, 광고 슬롯, AI 해석 기반이 있어요.
- 부족한 건 `AI 해석 이후의 연속 흐름`이에요.
- 그래서 이 PRD는 AI를 줄이는 문서가 아니라, **AI를 세션 확장 엔진으로 쓰는 문서**예요.

### 타겟 사용자
- 무료 운세나 무료 사주를 검색하는 모바일 사용자
- 표보다 말로 풀어주는 해석을 선호하는 사용자
- 한 번에 끝나는 계산보다 연애/직업/재물처럼 추가 질문을 하고 싶은 사용자
- 반복 방문형 운세 소비 습관이 있는 사용자

---

## 2. 제품 목표

### 비즈니스 목표
- 수익원은 광고만 유지해요.
- AI 해석으로 체류시간과 페이지뷰를 늘려 광고 노출을 키워요.
- 검색 유입과 재방문을 동시에 키워요.

### 제품 목표
- 계산 후 이탈을 줄여요.
- AI 해석 진입률을 높여요.
- AI 후속 행동을 자연스럽게 만들어요.
- 결과 페이지를 단일 페이지가 아닌 소비 흐름으로 바꿔요.

### 핵심 지표
- 계산 완료율
- AI 해석 진입률
- AI 후속 질문 클릭률
- 세션당 페이지 수
- 평균 참여 시간
- 재방문율
- 콘텐츠 -> 계산 전환율
- 계산 -> AI 전환율

## 3. 현재 상태 요약

### 확인된 강점
- AI 해석 CTA가 이미 존재해요. `src/components/App.tsx`
- 홈이 AI 해석 중심 메시지를 갖고 있어요. `src/components/Hero.tsx`, `index.html`
- AI 해석 API가 구현돼 있어요. `api/interpret.ts`
- 홈, 결과, 가이드, 랜딩에 광고 슬롯이 있어요. `src/components/App.tsx`, `src/pages/GuideIndex.tsx`, `src/pages/LandingPage.tsx`
- 다국어 라우팅과 가이드 구조가 있어요. `src/router.tsx`

### 확인된 약점
- AI 해석 이후의 흐름이 짧아요.
- 관련 글, 후속 질문, 재방문 CTA가 약해요.
- 기사 수가 적어요. `src/pages/ArticlesIndex.tsx`
- 측정 이벤트가 없어요.
- 사이트맵과 실제 확장 여지가 충분히 연결되지 않았어요. `public/sitemap.xml`

## 4. 핵심 기능

### P0

1. **첫 AI 해석 강화**
   - 계산 직후 바로 이해 가능한 AI 요약을 보여줘요.

2. **후속 질문 카드**
   - 사용자가 바로 누를 수 있는 질문 3~5개를 보여줘요.

3. **AI 해석 후 관련 글 추천**
   - 해석과 연결되는 가이드/아티클을 추천해요.

4. **재방문 CTA**
   - 오늘의 운세, 이번 주 조언, 다시 보기 진입점을 줘요.

5. **이벤트 추적**
   - 계산, AI, 콘텐츠, 광고 전환 흐름을 측정해요.

### P1

1. **오늘의 AI 운세**
2. **이번 주 AI 조언**
3. **공유용 AI 카드**
4. **콘텐츠와 AI 질문 자동 연결**

### P2

1. **템플릿형 운세 랜딩**
2. **다국어 AI 랜딩 확장**
3. **개인화된 재방문 홈**

### 재방문 흐름

## 6. 화면 전략

### 홈
- AI를 계속 메인 가치로 써요.
- “무료 계산”만이 아니라 “AI가 바로 풀어준다”를 전면에 둬요.
- CTA 하단에 `연애`, `직업`, `재물`, `성격` 같은 질문 예시를 붙여요.

### 결과 페이지
- 현재보다 AI 비중을 더 높여요.

권장 순서:
1. 결과 요약
2. AI 한 줄 해석
3. AI 상세 해석 버튼
4. 후속 질문 카드
5. 관련 글
6. 오늘의 운세 CTA
7. 광고

### 기사/가이드 페이지
- 본문 중간과 하단에 계산 CTA를 넣어요.
- 글 내용에 맞는 AI 질문을 연결해요.

예시:
- 십신 글이면 `내 사주의 십신은 어떻게 읽어야 해?`
- 오행 글이면 `내 오행 균형은 어떤 편이야?`

## 7. 기술 실현 가능성

| 항목 | 난이도 | 판단 |
|---|---|---|
| AI 후속 질문 카드 추가 | 쉬움 | 기존 모달/버튼 흐름 확장으로 가능해요 |
| 결과 페이지 재구성 | 보통 | `App.tsx` 중심 수정이 필요해요 |
| 오늘의 AI 운세 | 보통 | 입력값 재사용과 UI 설계가 필요해요 |
| 관련 글 자동 연결 | 보통 | 태그 맵 또는 규칙 기반 추천이 필요해요 |
| 이벤트 추적 | 보통 | 분석 스크립트와 이벤트 설계가 필요해요 |
| 기사 확장 | 쉬움 | 현재 구조 그대로 늘릴 수 있어요 |

### 주요 리스크
1. AI가 있어도 후속 행동이 없으면 광고 수익에 연결되지 않아요.
2. 광고가 AI 경험을 끊으면 체류시간이 줄어요.
3. 콘텐츠 수가 적으면 검색 유입이 생각보다 안 커져요.
4. 측정이 없으면 무엇이 먹히는지 못 알아요.

## 8. 수익 모델

### 원칙
- 수익은 광고만으로 만들어요.
- AI는 매출원이 아니라 광고 매출을 키우는 핵심 장치예요.

### 광고 배치 원칙
- AI보다 광고가 먼저 나오지 않아요.
- 광고는 흐름을 끊지 않는 위치에 둬요.
- AI 해석 직후와 관련 글 아래가 핵심이에요.

### 수익화 논리
- AI 해석 진입률 상승
- 후속 질문 클릭 상승
- 관련 글 클릭 상승
- 세션 길이 증가
- 광고 노출 수 증가
- 광고 수익 증가

## 9. 구현 로드맵

### Phase 1: AI 소비 흐름 만들기

기간: 1~2주

작업:
1. 결과 페이지에 AI 한 줄 요약 추가
2. 후속 질문 카드 3~5개 추가
3. AI 해석 후 관련 글 추천 추가
4. 추적 이벤트 설계

성공 기준:
- AI 진입률 상승
- AI 후속 클릭 발생
- 평균 참여 시간 상승

### Phase 2: 재방문 루프 만들기

기간: 2~4주

작업:
1. 오늘의 AI 운세 추가
2. 이번 주 AI 조언 추가
3. 최근 결과 다시 보기 강화
4. 공유 CTA 강화

성공 기준:
- 재방문율 상승
- 일간 운세 클릭 발생
- 공유 증가

### Phase 3: 검색 유입 확장

기간: 1~3개월

작업:
1. 기사 30개 이상 확대
2. 핵심 클러스터 운영
3. 기사와 AI 질문 연결
4. 다국어 랜딩 확장

성공 기준:
- 자연 검색 세션 증가
- 콘텐츠 -> 계산 전환 증가
- 계산 -> AI 전환 증가

## 12. 결론

명운판은 AI 비용을 줄이는 서비스가 아니에요.
명운판은 AI가 사용자를 더 오래 붙잡고, 더 많이 읽게 만들고, 다시 오게 만들어서 광고 수익을 키우는 서비스예요.
